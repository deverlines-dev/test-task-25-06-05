1. Причины выбора именно этого пакета composer для парсинга excel.

Есть хороший пакет - Laravel Excel, при меньших объемах можно и его выбрать.
FastExcel - будет быстрее и меньше ресурсов потребляет.
https://github.com/rap2hpoutre/fast-excel?tab=readme-ov-file#benchmarks
FastExcel основан на openspout/openspout - его и выбрал

2. Чем вы руководствовались, когда определяли правила валидации excel-файла.
Часто делал импорты, знаю о типичных ошибках, например:
Лишние данные в таблице
Неправильные наименования столбцов
Опечатки

По-хорошему нужно уточнить валидацию.
Применю такую:
id - string as numeric, не пустой
name - string, не пустой (каждое слово с большой буквы), не более 255 символов
date - d.m.Y, не пустой, валидная дата, не больше текущей
Можно добавить уникальность на id и name+date (спорный момент, нужно уточнить)
Ещё хорошо было бы поле name разделить на first_name и last_name (вероятно в дальнейшем пригодится, например для приветствия)
